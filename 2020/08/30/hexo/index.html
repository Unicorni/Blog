hexo主题matery使用记录

五款Hexo主题推荐：

1.Sakura 非常炫，缺点加载慢 https://github.com/honjun/hexo-theme-sakura
2.Matery 预加载，强推 https://github.com/blinkfox/hexo-theme-matery
3.3-hexo 三段式，适合博文多的 https://github.com/yelog/hexo-theme-3-hexo
4.Butterfly 也是个好看的 https://github.com/jerryc127/hexo-theme-butterfly/tree/dev
5.Next 功能集成多，简洁 https://github.com/theme-next/hexo-theme-next
hexo官网：https://hexo.io/

这篇文章主要记录一下matery使用、修改，本文操作基于你已经搭建好了node、hexo环境，并且含有一定英文基础或会使用搜索引擎

安装node、git、hexo可以查看这篇文章的前半部分

https://mp.weixin.qq.com/s/npQNr_ZgjNEH3h5IHNRgfg

要使用这个东西首先要看的自然是官方的中文文档啦

https://github.com/blinkfox/hexo-theme-matery/blob/develop/README_CN.md

0、初始化博客目录

在cmd或者terminal中cd到你想要的目录，例如，我想在E盘的Git目录下初始化：

E:\

cd Git

接下来输入命令

hexo init MyBlog

那么你的hexo博客已经搭建出来啦，可以本地测试一下

cd MyBlog

hexo s

访问终端显示的地址能够看到如下界面，就可以接着个性化你的博客啦

默认hexo主题

1、下载、切换主题

在终端使用ctrl+c结束上一条命令

输入以下命令下载主题

git clone https://github.com/blinkfox/hexo-theme-matery.git themes/matery

修改MyBlog目录下的hexo配置文件_config.yml

找到theme这一条，修改为matery

切换主题

此时运行hexo s，看到的就是matery主题了

matery

hexo配置文件的其他内容，自己修改，每个配置项官方文档列出来了

https://hexo.io/zh-cn/docs/configuration

2、修改主题配置文件

坐标MyBlog\themes\matery_config.yml

首先修改的是顶部的菜单栏

一级菜单

如果是你自己使用，当然是怎么方便怎么来，按照他上面的结构，复制一项，进行修改，可以直接使用中文，后面url跟的是页面所在的地址，icon是fontawesome图标

图标到这里找：https://fontawesome.com/icons

我们将前面#号去掉，二级菜单显示的默认还是英文，

二级菜单

我们需要在MyBlog\themes\matery\languages\zh-CN.yml中加入这些的翻译，对应默认的也加上英文，如果你上面写的说说为shuoshuo，也可以在这里添加，甚至你可以随便翻译，让它显示为你想要的内容

中文翻译

当然，我们自己用，应该不需要英文了吧，所以直接在配置文件中写中文就OK了

站点运行时间，这里提供了站点建立的时间，设置为true后将在网站下方显示

建立时间

首页封面轮播图，基本上不需要修改

轮播图

我的梦想，可以修改为自己想要的文字

音乐，提供了两种方案，第一种在页面左下角显示，第二种在我的梦想下方

音乐

我们这里主要修改的为show、id、fixed

show: 修改为自己想要的标题，例如欣赏音乐

id: 默认的是网易的，访问网页版网易云，登录后找到自己的歌单，在链接栏就会显示

网易id

视频不建议修改为true

首页推荐文章使用默认配置，之后会讲怎样让文章置顶

网站图标放在MyBlog/source下，命名为favicon.png

网站Logo放在MyBlog\themes\matery\source\medias下，名字为logo.png

首页按钮，可以修改为其他的，这里是GitHub，那么修改为自己的GitHub地址就行

社交图标

文章目录、代码相关建议不修改

打赏的将MyBlog\themes\matery\source\medias\reward下的两张图片换成自己的

复制追加版权信息建议设置为true

不需要使用公式的话，mathjax不建议设置为true

文章顶部显示的信息，建议都开启，字数统计需要安装插件

npm i –save hexo-wordcount

页面底部就会显示总字数，每篇文章中都会显示

由于接下来的配置涉及到一些还未添加的页面，配置暂时在这里停一下

3、新建页面

使用hexo new page “页面名”来创建一个页面

eg：新建tags，标签页

之后会在MyBlog/source中出现目录tags，修改里面的index.md

其他的类似

分类页 hexo new page categories

关于页 hexo new page “about”

留言板 hexo new page “contact”

友情链接 hexo new page “friends”

4、新建文章

开发者提供了这些可选项

可以先修改MyBlog/scaffolds/post.md

-–

title:

date: 1598783718994

author: Luwang

img: /source/images/xxx.jpg

keywords: 关键字

top: false

cover: true

coverImg: /images/1.jpg

password:

toc: true

mathjax: false

summary: 这是你自定义的文章摘要内容，如果这个属性有值，文章卡片摘要就显示这段文字，否则程序会自动截取文章的部分内容作为摘要

categories: 分类

tags:

- 标签1

- 标签2

reprintPolicy: cc_by

-–

之后每新建一篇文章都会调用这个模板
新建文章的命令为 hexo new 文章名

文章名建议使用英文，或者拼音。生成的文章会在source/_posts下,你可以进行修改

测试

5、页面详细配置

首页中的搜索栏，我们需要安装插件 npm install hexo-generator-search –save

RSS 安装插件 npm install hexo-generator-feed –save

修改hexo配置文件，在最底下添加

feed:

type: atom

path: atom.xml

limit: 20

hub:

content:

content_limit: 140

content_limit_delim: ‘ ‘

order_by: -date

标签和分类页，主要是文章的categories和tags字段

文章中的 安装代码高亮插件 npm i -S hexo-prism-plugin

修改hexo配置文件，highlight设置成false，添加

prism_plugin:

mode: ‘preprocess’ # realtime/preprocess

theme: ‘tomorrow’

line_number: false # default false

custom_css:

安装emoji表情插件 npm install hexo-filter-github-emojis –save

在hexo配置文件中加入

githubEmojis:

enable: true

className: github-emoji

inject: true

styles:

customEmojis:
